# Система анализа данных на основе дерева решений

Программная система для анализа табличных данных с применением методов построения **деревьев решений**.  
Проект разработан в рамках курсовой работы по теме:

> **«Разработка информационной системы для анализа данных с применением методов построения деревьев решений»**

Приложение реализовано в виде **Desktop App на Qt** и позволяет загружать наборы данных, выполнять их анализ, предобработку, обучение модели дерева решений, прогнозирование, оценку качества и визуализацию результатов.

## О проекте

Данная информационная система предназначена для автоматизации базового цикла анализа данных в задаче классификации.  
Пользователь может выбрать табличный файл с данными, указать целевой столбец и получить полный результат обработки:

- сводную информацию о наборе данных;
- подготовленный набор признаков;
- обученную модель дерева решений;
- прогнозы модели;
- метрики качества классификации;
- визуальное и текстовое представление дерева решений.

Проект построен по модульному принципу, что делает его удобным для сопровождения, тестирования и дальнейшего расширения.

---

## Цели системы

Основные цели разработки:

- реализовать минимальную, но полноценную информационную систему анализа данных;
- продемонстрировать применение дерева решений для задач классификации;
- предоставить понятный графический интерфейс для запуска полного аналитического сценария;
- обеспечить модульную архитектуру с разделением UI, бизнес-логики, доменных сущностей и инфраструктуры.

---

## Основные возможности

Система уже поддерживает следующий функционал:

- загрузка данных из файлов;
- автоматическое определение формата набора данных;
- анализ структуры набора данных;
- выделение числовых и категориальных признаков;
- обработка пропусков;
- кодирование категориальных признаков;
- разбиение данных на обучающую и тестовую выборки;
- обучение модели `DecisionTreeClassifier`;
- прогнозирование на `train`, `test` или `full`;
- расчет метрик качества:
  - Accuracy
  - Precision
  - Recall
  - F1-score
  - Confusion Matrix
- визуализация дерева решений:
  - в виде PNG-изображения;
  - в текстовом представлении;
- запуск полного сценария через единый контроллер приложения;
- покрытие ключевых модулей тестами.

## Архитектура системы

Проект организован по многоуровневому принципу и включает следующие слои:

### 1. `app`
Слой пользовательского приложения.  
Содержит:

- графический интерфейс на Qt;
- контроллер сценария работы;
- координацию пользовательских действий и вызовов бизнес-модулей.

### 2. `core`
Слой прикладной логики.  
Содержит основные сервисы обработки данных:

- загрузка;
- анализ;
- предобработка;
- построение дерева решений;
- прогнозирование;
- оценка качества;
- визуализация.

### 3. `domain`
Слой доменных сущностей.  
Описывает основные объекты предметной области:

- набор данных;
- обработанный набор данных;
- модель дерева решений;
- результаты прогнозирования;
- метрики оценки;
- данные визуализации.

### 4. `infrastructure`
Инфраструктурный слой.  
Содержит низкоуровневые технические реализации:

- чтение файлов разных форматов;
- определение формата файла;
- построение графического изображения дерева.

## Модули программы

В проекте выделены следующие программные модули:

### 1. Модуль пользовательского интерфейса (Qt Desktop App)
Обеспечивает взаимодействие пользователя с системой:
- выбор файла;
- выбор целевого столбца;
- запуск анализа;
- просмотр метрик;
- просмотр дерева решений.

### 2. Модуль управления сценарием работы
Координирует выполнение полного пайплайна обработки данных:
1. загрузка;
2. анализ;
3. предобработка;
4. обучение модели;
5. прогнозирование;
6. оценка;
7. визуализация.

### 3. Модуль загрузки данных
Отвечает за:
- проверку существования файла;
- определение формата;
- выбор соответствующего reader;
- чтение набора данных в `pandas.DataFrame`.

### 4. Модуль предварительной обработки данных
Выполняет:
- проверку целевого столбца;
- удаление строк без target;
- заполнение пропусков;
- one-hot encoding категориальных признаков;
- разбиение на train/test.

### 5. Модуль анализа данных
Формирует сводную информацию:
- число строк и столбцов;
- список признаков;
- числовые и категориальные столбцы;
- пропуски;
- дубликаты;
- распределение классов;
- числовые статистики.

### 6. Модуль построения дерева решений
Создает и обучает модель `DecisionTreeClassifier`, а также подготавливает сведения о:
- глубине дерева;
- количестве узлов;
- количестве листьев;
- важности признаков.

### 7. Модуль прогнозирования
Строит предсказания:
- на обучающей выборке;
- на тестовой выборке;
- на полном наборе данных;
- при наличии поддержки — вероятности классов.

### 8. Модуль оценки качества
Вычисляет основные метрики классификации:
- accuracy;
- precision;
- recall;
- f1-score;
- confusion matrix;
- распределение фактических и предсказанных классов.

### 9. Модуль визуализации
Формирует:
- текстовое представление дерева;
- PNG-изображение дерева;
- агрегированные данные для отображения в интерфейсе.

## Технологический стек

В проекте используются следующие технологии и библиотеки:

* **Python 3**
* **PySide6** — графический интерфейс на Qt
* **Pandas** — работа с табличными данными
* **Scikit-learn** — построение дерева решений и оценка качества
* **Matplotlib** — построение изображения дерева
* **Pytest** — тестирование
* **OpenPyXL / Excel backend** — чтение Excel-файлов

## Поддерживаемые форматы данных

Система поддерживает загрузку следующих форматов:

* `CSV`
* `TSV`
* `XLSX`
* `XLS`
* `JSON`

Определение формата выполняется автоматически на основе расширения файла.

## Сценарий работы приложения

Полный пользовательский сценарий выглядит так:

1. Пользователь выбирает файл с набором данных.
2. Система считывает данные и определяет список столбцов.
3. Пользователь выбирает целевой столбец.
4. Указывается режим прогнозирования:

   * `test`
   * `train`
   * `full`
5. Указывается стратегия усреднения метрик:

   * `weighted`
   * `macro`
   * `micro`
   * `binary`
6. Система запускает единый конвейер обработки:

   * загрузка;
   * анализ;
   * предобработка;
   * обучение модели;
   * прогнозирование;
   * оценка качества;
   * визуализация.
7. Результаты отображаются в графическом интерфейсе.

## Установка и запуск

### 1. Клонирование проекта

```bash
git clone https://github.com/akhmedovh4mid/DecisionTreeAnalyzer.git
cd DecisionTreeAnalyzer
```

### 2. Создание виртуального окружения

```bash
python -m venv .venv
```

### 3. Активация окружения

#### Windows

```bash
.venv\Scripts\activate
```

#### Linux / macOS

```bash
source .venv/bin/activate
```

### 4. Установка зависимостей

```bash
pip install -r requirements.txt
```

или, если используется `pyproject.toml`:

```bash
pip install -e .
```

### 5. Запуск приложения

```bash
python main.py
```

## Ключевые доменные сущности

Проект включает несколько центральных доменных сущностей:

### `Dataset`

Представляет исходный набор данных:

* имя;
* путь к файлу;
* таблица данных;
* формат;
* метаданные.

### `DatasetInfo`

Содержит аналитическую информацию о наборе данных:

* размерность;
* признаки;
* пропуски;
* дубликаты;
* распределение классов;
* числовую статистику.

### `ProcessedDataset`

Хранит результат предобработки:

* полный набор признаков;
* train/test выборки;
* целевой столбец;
* список признаков после кодирования.

### `DecisionTreeModel`

Инкапсулирует обученную модель дерева решений:

* sklearn-модель;
* признаки;
* классы;
* параметры;
* характеристики дерева.

### `PredictionResult`

Содержит результаты прогнозирования:

* входные признаки;
* предсказанные значения;
* реальные значения;
* вероятности классов.

### `EvaluationMetrics`

Содержит метрики качества модели:

* accuracy;
* precision;
* recall;
* f1-score;
* confusion matrix;
* распределения классов.

### `VisualizationData`

Содержит данные, необходимые для визуального отображения:

* текст дерева;
* путь к PNG-изображению;
* сводку по модели;
* сводку по метрикам;
* важность признаков.

## Особенности реализации

### Единая точка запуска бизнес-сценария

Модуль `ApplicationController` выступает координатором полного сценария анализа.
Это упрощает взаимодействие UI со всеми внутренними модулями.

### Разделение ответственности

Каждый модуль отвечает только за свою задачу:

* чтение файлов;
* анализ;
* предобработка;
* обучение;
* прогноз;
* оценка;
* визуализация.

### Поддержка категориальных и числовых признаков

В препроцессоре реализовано:

* заполнение пропусков;
* разделение по типам признаков;
* one-hot encoding категориальных столбцов.

### Подготовка визуализации

Дерево решений отображается:

* графически через `matplotlib`;
* текстово через `sklearn.tree.export_text`.

### Тестируемость

Ключевые бизнес-модули покрываются тестами, что позволяет контролировать корректность работы системы при расширении функциональности.
